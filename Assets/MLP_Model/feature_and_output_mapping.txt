NPC Interaction Policy Model Export Notes
========================================

Model name: policy_mlp_keep20
Input dim: 16
Output dim: 5

Input feature order:
Index 0: delta_s -> Relative longitudinal distance between target object and current NPC in Frenet s
Index 1: delta_d -> Relative lateral distance between target object and current NPC in Frenet d
Index 2: delta_v -> Relative speed between target object and current NPC
Index 3: euclid_dist -> Euclidean distance between target object and current NPC
Index 4: Speed -> Current NPC speed
Index 5: Throttle -> Current NPC throttle
Index 6: Brake -> Current NPC brake
Index 7: d_final -> Current NPC lateral offset from track centerline
Index 8: ds_dt -> Approximate longitudinal speed in Frenet coordinates
Index 9: dd_dt -> Approximate lateral speed in Frenet coordinates
Index 10: d2d_dt2 -> Approximate lateral acceleration in Frenet coordinates
Index 11: curvature -> Local track curvature
Index 12: turn_sign -> Turn direction sign
Index 13: left_margin -> Remaining free space to left track boundary
Index 14: right_margin -> Remaining free space to right track boundary
Index 15: track_width -> Local track width

Output action definition:
Output[0] = avoid_left -> NPC should move left to avoid nearby interaction risk
Unity behavior: Generate a smooth positive lateral offset in d
Output[1] = avoid_right -> NPC should move right to avoid nearby interaction risk
Unity behavior: Generate a smooth negative lateral offset in d
Output[2] = block_left -> NPC should move left to defend against approaching player
Unity behavior: Generate a smooth positive lateral offset in d
Output[3] = block_right -> NPC should move right to defend against approaching player
Unity behavior: Generate a smooth negative lateral offset in d
Output[4] = keep -> NPC should keep current line
Unity behavior: Keep current reference line, no extra lateral offset

Important runtime note:
1. Unity must build input vector in exactly the same feature order.
2. Missing values should be filled using imputer_statistics.
3. Standardize input using (x - mean) / scale.
4. ONNX output is classification logits, not continuous control values.
5. Use argmax(output) to get action index, then map to action label.
6. Convert action label to target delta_d, then update local x_ref/y_ref.
